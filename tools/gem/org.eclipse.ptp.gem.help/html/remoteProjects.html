Remote and Synchronized Projects

GEM supports remote and synchronized C/C++ MPI projects whose connections are through one of the following remote service providers:

  • Built-in SSH (default)
  • Remote Tools (now deprecated in PTP 8.0)
  • Remote System Explorer
Parallel Tools Platform (PTP),

Target machine requirements:

  • ISP version 0.3.0 or later
  • C/C++ Compiler
  • Some implementation of MPI must be installed, OpenMPI and MPICH2 both work well. If you are using MPICH v1.2 or lower, you will also need to have your process manager (e.g. mpd) and running. GEM assumes your process manager is already running when it is loaded. If you are using MPICH2 v1.3 or later with Hydra, you don't need to worry about this.

GEM operates and behaves identically with local, remote and synchronized C/C++ MPI projects. The only limitation is that GEM currently does not support the Happens Before Viewer with remote and synchronized projects, as it is an external tool. The Happens Before Viewer will soon be integrated into a new GEM view.
